# Разбиение датасета

В v1 используется разбиение 60/20/20: 72 train, 24 eval и 24 test.

Для каждого `question_id` есть пять вариантов качества ответа: `empty`, `weak`, `medium`, `good`, `strong`.

Текущее правило разбиения: `empty`, `weak`, `medium` попадают в train; `good` попадает в eval; `strong` попадает в test.

Такой вариант сохраняет полный набор уровней качества для каждого вопроса и остаётся удобным для первой итерации обучения. Если позже потребуется строгое 70/15/15, лучше увеличить число ответов на каждый вопрос до 6-7.
